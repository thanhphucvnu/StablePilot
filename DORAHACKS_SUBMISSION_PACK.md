# DoraHacks Submission Pack

Hackathon: Tether Hackathon Galactica: WDK Edition 1  
Submission page: https://dorahacks.io/hackathon/hackathon-galactica-wdk-2026-01/detail

## 1) What you need before clicking Submit

- DoraHacks account ready
- Project created on DoraHacks
- All teammates added to the project page
- Public code repository link (GitHub/GitLab/Bitbucket)
- Demo video link (max 5 minutes)
- Project description and technical explanation
- WDK integration clearly shown (mandatory)
- Third-party services/APIs list prepared
- Track selected

## 3) Required Assets Checklist

Mark each item done before final submit.

- [ ] Project name finalized
- [ ] One-line tagline prepared
- [ ] 150-300 word project overview written
- [ ] Problem statement and user pain explained
- [ ] Solution + architecture explained
- [ ] WDK integration details explained with code pointers
- [ ] Public repo URL added
- [ ] README includes setup/run instructions
- [ ] Demo video uploaded and shareable
- [ ] Video is 5 minutes or less
- [ ] Team intro and location filled
- [ ] Third-party dependencies disclosed
- [ ] Track selection confirmed
- [ ] Project page thumbnail/cover (if available) uploaded

## 4) Suggested Demo Video Structure (<= 5 min)

1. Intro (20-30s): team + problem
2. Solution overview (40-60s): what your app does
3. Live demo (2-3 min): key flow
4. Technical deep dive (60-90s): architecture + WDK usage
5. Closing (15-20s): impact + next steps

## 5) Judging Optimization Tips

- Show reliable execution, not just UI
- Emphasize autonomy and decision logic
- Explain economic soundness (costs, sustainability, risk controls)
- Prove real-world applicability with concrete use case
- Clearly demonstrate where WDK wallet/agent integration happens

## 6) Submission Day Runbook

1. Open DoraHacks project page
2. Paste finalized content from `submission/FORM_ANSWERS_TEMPLATE.md`
3. Add repo + demo links
4. Verify all teammates are attached
5. Re-open every link in incognito to confirm public access
6. Submit
7. Save screenshot/PDF of submitted page

## 7) Risk Checks Before Deadline

- Broken repo or private repo
- Video link permission errors
- Missing WDK proof
- Team members not attached on DoraHacks
- Missing third-party disclosure
- Last-minute edits not reflected on submission page

All hackathon-facing prose for StablePilot is in English in the files listed in section 2. After you record the demo, add the public video URL everywhere it appears as a placeholder and re-open links in a private/incognito window before submitting.
